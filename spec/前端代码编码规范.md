# 前端代码编码规范

## 文档说明
本规范适用于**纯前端 Vue 项目（无 Webpack/Vite 构建、直接原生运行、CDN 引入 Vue）**，前后端分离架构，遵循行业通用最佳实践，兼顾代码可读性、可维护性、协作效率和运行稳定性。

适用场景：直接通过浏览器打开 HTML 运行、无编译打包、无模块化构建、Vue  via CDN。

---

## 一、目录结构规范
### 1.1 标准目录结构
```
web/
├── index.html              # 项目入口（唯一主页面）
├── /assets                 # 静态资源（不编译，直接引用）
│   ├── /css                # 全局样式
│   │   └── global.css      # 全局通用样式
│   ├── /images             # 图片资源（png/jpg/svg/gif）
│   └── /icons              # 图标文件
├── /components             # 全局公共组件（.html/.js 双文件）
│   ├── UserCard.html       # 组件模板
│   └── UserCard.js         # 组件 JS 逻辑
├── /pages                  # 页面级组件
│   ├── Home.html
│   └── Login.html
├── /utils                  # 工具函数
│   └── request.js          # 原生 fetch/axios 请求封装
├── /config                 # 项目配置
    └── api.js              # 接口地址配置
```

### 1.2 目录使用规则
1. 禁止深层嵌套目录，最多 2 级嵌套，降低路径引用复杂度
2. 组件、页面统一使用**语义化名称**，禁止使用 `test/demo1` 等无意义命名
3. 静态资源禁止放在根目录，统一归类到 `assets`

## 二、命名规范
### 2.1 文件命名
1. **组件/页面文件**：大驼峰（PascalCase）
   - 正确：`UserList.html`、`LoginForm.js`
   - 错误：`userList.html`、`login-form.js`
2. **工具/配置文件**：小驼峰（camelCase）
   - 正确：`request.js`、`apiConfig.js`
3. **静态资源**：小写 + 短横线分隔
   - 正确：`logo-blue.png`、`bg-login.jpg`

### 2.2 变量/函数命名
1. **变量**：小驼峰，语义化，禁止缩写（通用缩写除外）
   - 正确：`userList`、`isLogin`、`tableLoading`
   - 错误：`ul`、`a`、`data`
2. **常量**：全大写 + 下划线
   - 正确：`MAX_COUNT`、`API_BASE_URL`
3. **函数**：小驼峰，**动词开头**
   - 正确：`getUserInfo()`、`handleSubmit()`、`openModal()`
4. **布尔值**：统一以 `is/has/can` 开头
   - 正确：`isVisible`、`hasPermission`

### 2.3 样式类名
1. 小写 + 短横线分隔
2. 语义化，禁止泛义类名
   - 正确：`user-card`、`form-item`
   - 错误：`blue`、`big`、`a1`

## 三、HTML 规范（Vue 模板）
### 3.1 基础格式
1. 缩进统一使用 **2 个空格**
2. 标签必须闭合，自闭合标签添加 `/`
3. 属性顺序固定：`id` → `class` → `v-指令` → `原生属性` → `事件`

```html
<!-- 正确 -->
<div
  id="user-card"
  class="user-card"
  v-if="isShow"
  v-for="item in list"
  :key="item.id"
  title="用户卡片"
  @click="handleClick"
>
</div>
```

### 3.2 Vue 指令规范
1. `v-for` 必须搭配 `:key`，key 使用唯一 id，禁止用 index
2. `v-for` 与 `v-if` 禁止同标签使用
3. 指令缩写统一使用：
   - `:property` 代替 `v-bind:property`
   - `@event` 代替 `v-on:event`

```html
<!-- 错误 -->
<div v-for="item in list" v-if="item.active"></div>

<!-- 正确 -->
<template v-if="isShow">
  <div v-for="item in list" :key="item.id"></div>
</template>
```

### 3.3 无构建项目特殊规则
1. 不使用单文件组件（`.vue`），模板写在 `.html` 文件中
2. 不使用 `import/require`，通过全局变量挂载组件
3. CDN 资源放在 `<head>` 末尾，业务脚本放在 `<body>` 末尾

## 四、JavaScript 规范（Vue 原生写法）
### 4.1 代码格式
1. 缩进 4 个空格
2. 语句末尾必须加分号
3. 字符串统一使用单引号 `''`
4. 对象最后一个属性后保留逗号（便于版本管理）

### 4.2 Vue 选项顺序（强制）
```js
const App = {
  // 1. 组合式函数（若使用 Vue3）
  setup() {},
  
  // 2. 核心属性
  name: 'App',
  components: {},
  props: {},

  // 3. 数据
  data() {
    return {}
  },
  computed: {},

  // 4. 监听
  watch: {},

  // 5. 生命周期
  created() {},
  mounted() {},

  // 6. 方法
  methods: {}
}
```

### 4.3 数据与方法
1. `data` 必须是函数，返回对象
2. 禁止在模板中写复杂逻辑，复杂逻辑抽离为 `computed`
3. 异步函数统一使用 `async/await`，禁止多层回调嵌套

```js
// 正确
async getUserList() {
  try {
    this.loading = true
    const res = await fetchUserList()
    this.userList = res.list
  } catch (err) {
    console.error(err)
  } finally {
    this.loading = false
  }
}
```

### 4.4 注释规范
1. 文件头部必须添加注释：作者、功能、创建时间
2. 复杂函数添加块注释，说明参数、返回值、功能
3. 关键逻辑行尾添加单行注释

```js
/**
 * 获取用户列表
 * @param {Number} page - 页码
 * @param {Number} size - 每页条数
 * @return {Array} 用户列表
 */
async getUserList(page, size) {}
```

## 五、CSS/样式规范
### 5.1 基础规则
1. 缩进 2 个空格
2. 样式属性顺序：布局 → 尺寸 → 文本 → 视觉
3. 禁止使用 `!important`（除非覆盖第三方样式）

### 5.2 作用域规范
无构建版无法使用 `scoped`，必须通过**父级类名做样式隔离**：
```css
/* 正确 */
.user-card .title {
  font-size: 16px;
}

/* 错误 */
.title {
  font-size: 16px;
}
```

### 5.3 通用规范
1. 颜色使用十六进制，不使用色值名称
2. 单位统一：`px`（固定）、`%`（自适应）
3. 公共样式抽离到 `global.css`，禁止重复编写

## 六、API/网络请求规范
### 6.1 请求封装
统一封装请求工具，统一处理：请求头、错误、加载状态、超时

```js
// utils/request.js
const API_BASE_URL = 'https://api.xxx.com'

async function request(url, options = {}) {
  const token = localStorage.getItem('token') || ''
  try {
    const response = await fetch(API_BASE_URL + url, {
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${token}`
      },
      ...options
    })
    const data = await response.json()
    if (data.code !== 200) throw new Error(data.msg)
    return data
  } catch (err) {
    alert(err.message || '请求失败')
    throw err
  }
}
```

### 6.2 接口管理
所有接口统一维护在 `config/api.js`，禁止硬编码

```js
// config/api.js
const API = {
  USER_LIST: '/user/list',
  LOGIN: '/user/login'
}
```

## 七、组件开发规范
### 7.1 组件定义方式
模板 + JS 分离，通过全局注册使用

```html
<!-- components/UserCard.html -->
<template id="user-card">
  <div class="user-card">{{ user.name }}</div>
</template>
```

```js
// components/UserCard.js
const UserCard = {
  template: '#user-card',
  props: {
    user: {
      type: Object,
      required: true
    }
  }
}

// 全局注册
Vue.component('UserCard', UserCard)
```

### 7.2 组件通信
1. 父子：`props` + `$emit`
2. 跨级/全局：`eventBus`（无 Vuex/Pinia）

```js
// 全局 eventBus
const bus = new Vue()

// 发送
bus.$emit('refresh', data)

// 接收
bus.$on('refresh', (data) => {})
```

## 八、版本管理与提交规范

### 8.1 提交信息格式
`type: description`
- `feat`：新功能
- `fix`：修复 bug
- `style`：样式/格式调整
- `refactor`：重构
- `docs`：文档

示例：
```bash
feat: 新增用户列表组件
fix: 修复登录请求异常
```

## 九、安全与性能规范
1. 禁止明文存储密码、密钥，使用 `localStorage` 存储 token
2. 请求必须处理异常，避免页面崩溃
3. 图片统一压缩，禁止使用超大图
4. `v-for` 必须用 `key`，提升渲染性能
5. 不滥用全局变量，避免命名冲突
6. 敏感接口添加权限校验

---

## 十、调试与错误处理
1. 禁止上线保留 `console.log`
2. 异步请求必须 `try/catch`
3. 错误信息统一提示，不原生暴露
4. 禁止使用 `alert` 做正式环境提示（可封装轻量提示）

---

## 十一、强制校验规则（团队协作）
1. 所有文件必须遵循本规范
2. 禁止使用未定义的变量
3. 禁止修改他人组件，如需修改走合并流程
4. 公共组件必须添加注释和使用示例
5. 禁止内联样式，禁止行内复杂逻辑

---

# 总结
1. 适配**直接运行、无编译打包、CDN 引入**的项目模式
2. 结构清晰、命名统一、代码格式强制标准化，降低协作成本
3. 组件、请求、样式、安全、性能全覆盖，可直接作为团队文档使用
4. 完全兼容原生 Vue 写法，不依赖任何构建工具，开箱即用
