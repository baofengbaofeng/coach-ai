# 代码审查工作指南

## 📋 概述

本文档是CoachAI项目的代码审查工作指南，由CR (Code Review) subagent创建和维护。文档按照项目文档规范要求，使用中文文件名，存储在`docs/开发指南/`目录下。

## 🎯 代码审查目标

### 1. 质量目标
- **代码质量**: 确保代码符合项目编码规范和质量标准
- **安全质量**: 识别和预防安全漏洞和风险
- **性能质量**: 优化代码性能，避免性能陷阱
- **可维护性**: 提高代码可读性和可维护性

### 2. 团队目标
- **知识共享**: 通过审查促进团队知识共享
- **标准统一**: 统一团队编码标准和实践
- **能力提升**: 提升团队代码质量和安全意识
- **协作改进**: 改进团队协作和沟通效率

## 📝 审查工作流程

### 1. 审查准备阶段
#### 1.1 获取审查代码
- 从Git仓库获取待审查的代码
- 确认代码变更范围和影响
- 了解相关业务需求和技术方案

#### 1.2 环境准备
- 配置代码审查工具和环境
- 准备审查标准和检查清单
- 设置审查记录和报告模板

### 2. 审查执行阶段
#### 2.1 规范检查
- 检查代码是否符合编码规范
- 验证命名规范、注释规范、格式规范
- 确认导入规范、类型注解规范

#### 2.2 质量检查
- 检查代码逻辑正确性和完整性
- 验证错误处理和异常处理
- 评估代码性能和资源使用

#### 2.3 安全检查
- 检查安全漏洞和风险
- 验证输入验证和数据验证
- 评估权限控制和访问控制

#### 2.4 可维护性检查
- 评估代码结构和模块设计
- 检查代码复杂度和耦合度
- 验证测试覆盖率和质量

### 3. 报告生成阶段
#### 3.1 问题记录
- 详细记录发现的问题
- 分类整理问题类型和严重程度
- 提供具体的改进建议

#### 3.2 报告编写
- 生成标准化的审查报告
- 包含问题汇总和改进建议
- 提供优先级和修复建议

#### 3.3 报告存储
- 按照文档规范存储报告
- 使用中文文件名，存储在docs目录
- 纳入Git版本控制管理

### 4. 跟踪改进阶段
#### 4.1 问题跟踪
- 跟踪问题的修复情况
- 验证修复效果和质量
- 更新问题状态和记录

#### 4.2 经验总结
- 总结审查经验和教训
- 更新审查标准和检查清单
- 分享最佳实践和改进建议

## 🔧 审查工具和方法

### 1. 静态分析工具
#### Python代码审查工具：
- **pylint**: 代码质量检查
- **flake8**: 编码规范检查
- **mypy**: 类型检查
- **bandit**: 安全漏洞检查
- **black**: 代码格式化检查

#### 前端代码审查工具：
- **ESLint**: JavaScript/TypeScript代码检查
- **Prettier**: 代码格式化
- **TypeScript编译器**: 类型检查
- **安全扫描工具**: 安全漏洞检查

### 2. 代码质量指标
#### 量化指标：
- **代码复杂度**: 圈复杂度、认知复杂度
- **代码重复率**: 重复代码检测
- **测试覆盖率**: 单元测试覆盖率
- **缺陷密度**: 每千行代码缺陷数

#### 质量评分：
- **规范符合度**: 编码规范符合程度
- **可读性评分**: 代码可读性评估
- **可维护性评分**: 代码可维护性评估
- **安全性评分**: 代码安全性评估

### 3. 审查检查清单
#### 通用检查清单：
- [ ] 编码规范符合性
- [ ] 命名规范一致性
- [ ] 注释文档完整性
- [ ] 错误处理完善性
- [ ] 测试覆盖充分性

#### 安全检查清单：
- [ ] 输入验证和过滤
- [ ] SQL注入防护
- [ ] XSS攻击防护
- [ ] 权限控制验证
- [ ] 敏感信息保护

#### 性能检查清单：
- [ ] 算法复杂度优化
- [ ] 资源使用效率
- [ ] 数据库查询优化
- [ ] 缓存策略合理
- [ ] 并发处理安全

## 👥 团队协作机制

### 1. 审查沟通流程
#### 问题反馈：
- 通过审查报告反馈问题
- 提供具体的改进建议
- 标注问题优先级和严重程度

#### 问题讨论：
- 组织代码审查会议
- 讨论复杂问题和解决方案
- 确定改进计划和时间表

#### 问题跟踪：
- 使用问题跟踪系统
- 跟踪问题修复进度
- 验证修复效果和质量

### 2. 知识共享机制
#### 最佳实践分享：
- 定期分享代码最佳实践
- 总结审查经验和教训
- 更新审查标准和指南

#### 培训和教育：
- 组织代码规范培训
- 分享安全编码实践
- 提供性能优化指导

#### 文档维护：
- 维护代码审查文档
- 更新审查标准和检查清单
- 分享成功案例和经验

- **知识共享**: 最佳实践分享 > 2次/月
- **能力提升**: 团队能力提升评分 > 4.0/5.0

### 3. 过程成功标准

- **持续改进**: 改进措施实施率 > 80%
