# Coach AI 产品经理工作指南

## 🎯 产品经理职责

### 核心职责
1. **需求管理**
   - 收集和分析用户需求
   - 编写和维护需求文档
   - 管理需求优先级和变更

2. **产品规划**
   - 制定产品路线图和迭代计划
   - 定义产品功能和用户体验
   - 管理产品Backlog和发布计划

3. **跨部门协作**
   - 与开发团队协作，确保需求理解一致
   - 与设计团队协作，定义用户体验
   - 与运营团队协作，制定推广策略
   - 与测试团队协作，确保产品质量

4. **数据分析**
   - 监控产品关键指标
   - 分析用户行为数据
   - 基于数据优化产品决策

5. **市场研究**
   - 跟踪竞品动态
   - 分析市场趋势
   - 识别商业机会

## 📋 工作流程

### 每日工作
1. **晨会** (9:00-9:15)
   - 同步团队进度
   - 识别问题和风险
   - 调整当日计划

2. **需求处理** (9:15-11:30)
   - 处理用户反馈
   - 更新需求文档
   - 与开发团队沟通需求细节

3. **数据分析** (13:30-15:00)
   - 查看产品数据报表
   - 分析用户行为
   - 识别优化机会

4. **跨部门会议** (15:00-16:30)
   - 与设计团队评审界面设计
   - 与运营团队讨论推广计划
   - 与测试团队确认验收标准

5. **文档更新** (16:30-17:30)
   - 更新产品文档
   - 编写需求说明
   - 准备次日工作

1. **周一**: 周计划制定

2. **周二**: 需求评审会
   - 评审新需求
   - 确定开发优先级
   - 分配开发任务

3. **周三**: 产品设计评审
   - 评审界面设计
   - 确认交互流程
   - 优化用户体验

4. **周四**: 数据分析日
   - 深度数据分析
   - 用户调研分析
   - 竞品分析更新

5. **周五**: 周总结和规划
   - 总结本周工作
   - 规划下周重点
   - 团队知识分享

### 每月工作
1. **月初**: 月度计划
   - 制定月度目标
   - 更新产品路线图
   - 资源规划和分配

2. **月中**: 进度检查

3. **月末**: 月度总结

## 🎯 优先级管理方法

### 优先级评估模型
使用RICE评分模型评估需求优先级:

1. **Reach (覆盖范围)**
   - 影响多少用户
   - 使用频率如何
   - 评分: 1-10分

2. **Impact (影响力)**
   - 对用户的价值
   - 对业务的价值
   - 评分: 0.25-3分

3. **Confidence (信心度)**
   - 需求确定性
   - 数据支持程度
   - 评分: 50%-100%

4. **Effort (投入成本)**
   - 开发工作量
   - 设计工作量
   - 测试工作量
   - 评分: 人天数量

**计算公式**: (Reach × Impact × Confidence) ÷ Effort

### 优先级分类
1. **P0 - 必须**: 核心功能，没有则产品无法使用
2. **P1 - 重要**: 重要功能，显著提升用户体验
3. **P2 - 期待**: 增强功能，提升产品竞争力
4. **P3 - 未来**: 扩展功能，为未来发展准备

### 优先级调整原则
1. **用户价值优先**: 优先解决用户痛点
2. **商业价值导向**: 考虑功能对收入的贡献
3. **技术可行性**: 考虑实现难度和风险
4. **资源约束**: 考虑团队能力和时间
5. **战略对齐**: 符合产品长期发展方向

## 📊 数据分析方法

### 关键指标监控
1. **用户增长指标**
   - 新增用户数
   - 活跃用户数
   - 用户留存率
   - 用户流失率

2. **用户行为指标**
   - 功能使用率
   - 用户参与度
   - 用户满意度
   - 用户反馈评分

3. **商业指标**
   - 收入指标
   - 成本指标
   - 盈利指标
   - 投资回报率

4. **产品性能指标**
   - 系统可用性
   - 响应时间
   - 错误率
   - 数据质量

### 数据分析方法
1. **趋势分析**: 分析指标变化趋势
2. **对比分析**: 对比不同用户群体
3. **漏斗分析**: 分析用户转化路径
4. **留存分析**: 分析用户留存情况
5. **细分分析**: 分析不同用户细分

### 数据驱动决策
1. **假设验证**: 基于数据验证产品假设
2. **效果评估**: 评估功能上线效果
3. **优化方向**: 基于数据确定优化方向
4. **风险预警**: 基于数据识别风险

## 🤝 团队协作指南

### 与开发团队协作
1. **需求沟通**
   - 提供清晰的需求文档
   - 解释业务背景和用户场景
   - 回答技术实现相关问题

2. **进度同步**
   - 每日站会同步进度
   - 及时识别和解决问题
   - 调整计划和优先级

3. **质量保证**
   - 定义明确的验收标准
   - 参与功能测试和验收
   - 收集用户反馈并优化

### 与设计团队协作
1. **设计需求**
   - 提供详细的产品需求
   - 解释用户场景和使用流程
   - 定义设计目标和原则

2. **设计评审**
   - 评审设计方案的合理性
   - 确认设计符合产品目标
   - 优化用户体验细节

3. **设计验收**
   - 验收设计稿的完整性
   - 确认设计实现效果
   - 收集用户对设计的反馈

### 与运营团队协作
1. **产品推广**
   - 提供产品核心价值点
   - 制定推广策略和计划
   - 准备推广素材和内容

2. **用户运营**
   - 分析用户行为和需求
   - 制定用户运营策略
   - 优化用户生命周期管理

3. **数据运营**
   - 定义数据监控指标
   - 分析运营效果数据
   - 优化运营策略和方案

### 与测试团队协作
1. **测试需求**
   - 提供详细的测试需求
   - 定义功能验收标准
   - 解释业务逻辑和场景

2. **测试协作**
   - 参与测试用例评审
   - 协助复现和定位问题
   - 确认问题修复效果

3. **质量评估**
   - 评估产品质量水平
   - 分析用户反馈问题
   - 制定质量改进计划

## 📈 产品成功评估

1. **用户维度**
   - 用户满意度
   - 用户留存率
   - 用户推荐率
   - 用户使用时长

2. **产品维度**
   - 功能完整性
   - 用户体验
   - 系统稳定性
   - 产品质量

3. **商业维度**
   - 用户增长

   - 盈利能力

4. **市场维度**

2. **定性评估**: 用户调研，反馈分析
3. **对比评估**: 竞品对比，市场对比
4. **趋势评估**: 时间趋势，发展预测

### 评估周期
1. **日常评估**: 每日监控关键指标
2. **周度评估**: 每周分析产品表现
3. **月度评估**: 每月总结产品成果
